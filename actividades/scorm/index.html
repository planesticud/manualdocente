    SCORM

    El paquete SCORM (Shareable Content Object Reference Model, o en español como Modelo de Referencia para Objetos de Contenido), son un paquete de archivos para los objetos de aprendizaje. Aquí se permite agregar paquetes SCORM como archivos zip para su futura visualización en el aula.

    El contenido es interactivo entre páginas de contenido con navegación entre ellas. Estos paquetes por lo general, cuentan con preguntas calificables y que se pueden registrar en el libro de calificaciones del curso.

    Las actividades SCORM se puede usar

    • Para la presentación de contenidos multimedia y animaciones
    • Como herramienta de evaluación

    Para cargar un paquete SCORM se debe seguir la siguiente ruta: Activar la edición del curso en el botón “Activar edición” luego, ir al Tema, Unidad, pestaña donde se debe reflejar el módulo y dar clic en “Añadir una actividad o recurso” buscar en el menú de “Actividades” el nombre de “Paquete SCORM”

    Imagen 79. Activar edición del curso.

    Imagen 80. Añadir una actividad o recurso.

    Imagen 81. Elección de actividad Paquete SCORM.

    Al crear la actividad nos redirecciona al menú de configuración de este, donde debemos agregar el nombre, apariencia, parámetros, entre otros ajustes. Se indicará uno a uno los apartados a configurar.

    • General: En los ajustes generales, nos solicita un nombre (que es obligatorio para la creación de cualquier recurso o actividad), una descripción, si así se requiere.

    Imagen 82. Configuración apartado General en actividad Paquete SCORM.

    • Paquete: Aquí es donde cargaremos el paquete SCORM que queremos usar para el curso. Este paquete debe ser ZIP.

    Imagen 83. Cargue de paquete SCORM.

    • Apariencia: Aquí se configura la forma de mostrar el paquete en el curso, puede ser por una ventana emergente o en la misma página del curso, así como mostrar la navegación y estado de intentos, entre otros apartados.

    Imagen 84. Configuración de apariencia en Paquete SCORM.

    • Disponibilidad: Aquí se configura la disponibilidad en tiempo que tendrá la actividad, configurando la fecha desde que se encuentra disponible hasta la fecha final.

    Imagen 85. Configuración de disponibilidad en Paquete SCORM.

    • Calificación: Para configurar la calificación, se debe elegir el método de calificación, entre las opciones se encuentran: calificación más alta, promedio, sumadas o del objeto de aprendizaje; y su calificación máxima.

    Imagen 86. Configuración de calificación de Paquete SCORM.

    • Gestión de intentos: Se puede configurar lo referente a los intentos, como el número de intentos, la calificación de estos, si se debe forzar uno nuevo o incluso bloquear después del último intento..

    Imagen 87. Configuración apartado Gestión de intentos en Paquete SCORM.

    • Configuración de compatibilidad: Aquí se configura el forzar la finalización de la actividad, así como la continuación automática de donde queda en la última visualización y un autoguardado..

    Imagen 88. Configuración apartado Gestión de intentos en Paquete SCORM.

    • Ajustes comunes del módulo: Aquí podremos elegir si se le mostrará el paquete SCORM en el curso a los estudiantes o está se ocultará y será visible solo para los profesores, de manera predeterminada en su rol. Además de agregar un número ID para dar una identificación para fines de informes personalizados. Este espacio se puede dejar en blanco, como viene de forma predeterminada. Adicional, se puede asignar el modo de grupo y agrupamiento, así como añadir una nueva restricción.

    Imagen 89. Configuración de ajustes comunes del módulo en Paquete SCORM.

    • Restricción de acceso: Aquí podremos restringir el acceso según la configuración que asignemos, esta puede ser por finalización de una actividad en específico, fecha determinada, calificación de una actividad en específico, perfil de usuario, incluso un conjunto de restricciones .

    Imagen 90. Configuración apartado Restricción de acceso en Paquete SCORM.

    • Finalización de actividad: En este ítem se puede configurar como el estudiante marcará o obtendrá la finalización de la actividad si esta se requiere, además de una fecha de espera de finalización del recurso carpeta (más una fecha de revisión que finalización).

    Imagen 91. Configuración apartado Finalización de actividad en Paquete SCORM.
